# Memex CLI 契约

> 日期：2026-06-01
> 作用范围：`memex` 二进制（`crates/memex-cli`）的 stdout / stderr / exit code / `--json` 输出形状。
> 用途：脚本化集成、CI 检测、Tauri/MCP 兼容。

---

## 1. 全局约定

| 维度 | 行为 |
|---|---|
| `--json` 全局 flag | 所有命令支持。打开后 stdout 输出严格 JSON（pretty），关闭则输出人类可读文本。 |
| stdout | 仅放命令"结果"（数据 / JSON），不混杂日志。 |
| stderr | 放运行日志（`tracing` 输出，由 `RUST_LOG` 控制）+ 显式 `eprintln!` 提示。 |
| exit code | 当前仅区分 0（成功）与非 0（失败）。失败由 `anyhow::Error` 通过 `main` 默认通道打印到 stderr。 |
| 中断（Ctrl-C） | exit code 130（SIGINT 默认）。不做额外处理。 |
| 输入参数错误 | clap 输出帮助到 stderr，exit code 2。 |

> **未来扩展**：v0.4+ 可引入分级 exit code（参见 §5 候选方案）。当前所有"业务失败"都返回 1。

---

## 2. 子命令契约总览

| 命令 | 默认 stdout | `--json` 输出 | 关键 stderr | exit |
|---|---|---|---|---|
| `memex ingest [--adapter X]` | 进度行（每个 adapter 一条 summary） | `{"adapter":"...","new_sessions":N,"new_messages":M,"errors":[]}` 数组 | adapter 错误 warn | 0 / 1 |
| `memex search <q>` | 高亮结果块 | `{"query":"...","latency_ms":N,"results":[SearchResult]}` | "no daemon, falling back to direct" | 0 |
| `memex sessions [--recent N] [--days N]` | 表格 | `{"sessions":[SessionRow]}` | 无 | 0 |
| `memex session <id>` | 完整 markdown 渲染 | `SessionDetail` | "session not found" | 0 / 1 |
| `memex stats` | 多行汇总 | `{"sessions":N,"messages":N,"chunks":N,"summaries":N,"today":{...},"last_7d":[...]}` | 无 | 0 |
| `memex config show` | KEY = VAL 行 | `MemexConfig` 序列化 | 无 | 0 |
| `memex config set <k> <v>` | `set <k> = <v>` | `{"key":"...","value":"...","saved":true}` | "unknown key" → stderr | 0 / 1 |
| `memex doctor` | 多段报告 | `DoctorReport` | 无 | 0 |
| `memex backup <path>` | "wrote N bytes to PATH" | `{"path":"...","bytes":N}` | 无 | 0 / 1 |
| `memex rebuild-index` | 进度行 | `{"sessions_rebuilt":N,"messages":N,"chunks":N}` | warn 损坏文件 | 0 / 1 |
| `memex mcp` | （无，stdio JSON-RPC） | 同左 | tracing 日志 | 由 client 关闭决定 |
| `memex setup <target>` | "wrote ~/.cursor/mcp.json" | 无（暂不支持 `--json`） | "Unknown target: X" → stderr | 0 |
| `memex daemon start\|stop\|status` | 单行状态 | `{"running":bool,"pid":N,"port":N}` | 无 | 0 / 1 |
| `memex credentials set/show/clear` | 单行 | `{"provider":"...","status":"set\|cleared"}` | "unsupported provider" → stderr | 0 |

> **SearchResult / SessionRow / SessionDetail / MemexConfig / DoctorReport** 的精确字段定义见 `crates/memex-core/src/storage/json_contract_tests.rs` 与 `tauri-app/src/types/index.ts`。两者必须保持一致，由 IPC contract 测试守护。

## 3. stderr / 日志渠道说明

* `tracing` 通过 `RUST_LOG` 控制（默认 `info`），通过 `tracing-subscriber` 写到 **stderr**。
* `eprintln!` 用于显式的用户提示（"unknown target"、"unsupported provider"），独立于 tracing 级别。
* `println!` 仅 stdout，且必须满足上表"默认 stdout"或"--json 输出"两种之一。

**示例**：脚本只关心结果时，丢弃 stderr：
```bash
memex search redis --json 2>/dev/null | jq '.results | length'
```

调试 adapter 行为时，提升日志：
```bash
RUST_LOG=memex=debug memex ingest --adapter cursor 2>err.log
```

## 4. 失败模式与 exit code

当前实现：

| 场景 | 行为 | exit code |
|---|---|---|
| 命令行参数缺失 / 类型错 | clap 帮助 → stderr | 2 |
| 数据库未初始化 | 返回"empty" 结果（如 `stats: 0/0/0/0`），不报错 | 0 |
| 数据库存在但读失败（损坏） | anyhow 错误 → stderr | 1 |
| Adapter 单个 collector 失败 | warn → stderr，继续 | 0 |
| 网络（cloud LLM）失败 | warn → stderr，本次摘要 skip | 0 |
| `memex daemon start` 端口被占 | "daemon already running on port N" → stderr | 1 |
| `memex daemon stop` 无运行 | "no daemon found" → stderr | 1 |
| `memex backup <path>` 路径不可写 | anyhow 错误 → stderr | 1 |
| `memex rebuild-index` 找不到 markdown 真源 | "no sessions to rebuild" → stdout，exit 0 | 0 |

## 5. 未来扩展：分级 exit code（候选，未落地）

| code | 含义 |
|---|---|
| 0 | 成功 |
| 1 | 通用业务失败（DB / IO / 序列化） |
| 2 | 参数错误（clap 已用此值） |
| 10 | adapter 全部失败 |
| 11 | 配置文件不可读 / 不可写 |
| 12 | daemon 操作失败（端口冲突 / lockfile 残留） |
| 20 | LLM provider 全部不可用（但 `ingest` 仍 exit 0，不阻塞确定性步骤） |
| 130 | SIGINT（继承 shell 约定，不重复发明） |

如果决定落地，应同时给 `memex --json` 加 `"exit_reason":"..."` 字段，方便 CI 不解析 stderr 也能定位失败类型。

---

## 6. 兼容性保证

* 任何 `--json` 输出字段的删除 / 重命名视为 **breaking change**，必须升 minor 版本。
* 默认 human 输出格式 **不属于** 契约，可以任意改。
* 由 `crates/memex-core/src/storage/json_contract_tests.rs` 守护核心 `SearchResult` / `SessionRow` / `SessionDetail` / `StatsBreakdown` / `TimelineEntry` / `ProjectSummary` / `Stats` 的字段稳定性。
* Tauri IPC（`tauri-app/src-tauri/tests/ipc_contract.rs`）守护 menubar / dashboard 用到的 struct 字段。

---

## 7. 维护检查清单

每次增删 `--json` 字段时：

1. 修改对应 `Db::...` 查询和返回 struct。
2. 同步 `crates/memex-core/src/storage/json_contract_tests.rs` 中的断言。
3. 同步 `tauri-app/src/types/index.ts` 中的 TS 类型。
4. 同步 `tauri-app/src-tauri/tests/ipc_contract.rs` 中的字段名断言。
5. 更新本文件 §2 表格。
6. 在 commit message 中标明 `BREAKING:` 前缀（如适用）。
