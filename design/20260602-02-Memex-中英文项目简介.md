> A local-first session-memory hub that indexes conversation history from Cursor, Claude Code, Codex, OpenCode, Aider, Continue and Cline into a single SQLite + FTS5 store and exposes it to any AI editor over MCP. macOS menubar app, offline by default.

### Short paragraph (README / About section, ~150 words)

> **Memex** is a local-first memory hub for AI coders. It silently watches the session files of 7 popular AI editors—Cursor, Claude Code, Codex, OpenCode, Aider, Continue, and Cline—normalises and redacts the messages, then full-text indexes them into a single local SQLite + FTS5 store. The corpus is exposed over the standard MCP protocol, so every new session in any AI editor can "remember" what you discussed three months ago in a different tool.
>
> Everything stays on your laptop. Nothing is uploaded by default; the optional cloud summarisation path requires explicit opt-in and runs through a redaction pipeline first.
>
> Ships as a macOS menubar app with a ⌘⇧M global shortcut, a popup search, a full dashboard window, and one-click MCP/SKILL deployment to all 4 supported IDEs.

### 3-bullet value prop (landing / HN post / tweet thread)

> 1. **Unify** — 7 AI coding tools (Cursor, Claude Code, Codex, OpenCode, Aider, Continue, Cline), one searchable corpus.
> 2. **Local** — everything lives in `~/.memex/`, FTS5-indexed, nothing uploaded by default.
> 3. **Embeddable** — speaks MCP, so any AI editor instantly gains `search_memory` / `get_session` tools over your entire AI history.

### Tribute line (footer / tagline, optional)

> A tribute to Vannevar Bush's 1945 *As We May Think*—the Memex he imagined 80 years ago, finally built for the AI era.
