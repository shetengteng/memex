## MCP（Rust 内置）

| crate | 用途 |
|---|---|
| `rmcp`（首选）/ 手写 stdio | MCP stdio 协议处理 |
| `tokio` | async I/O |
| `serde_json` | tool call 协议序列化 |

MCP 不再是独立进程，而是 `memex-cli` 的一个子命令：`memex mcp` 进入 stdio 模式后开始监听 MCP 协议。所有 tool call（search / get_session / recent_sessions / stats）直接调用 `memex-core` 函数，不再 spawn 子进程，启动开销 < 10ms。

## 架构约束

1. **Markdown 是真源** — SQLite 可从 Markdown 重建
2. **CLI 是稳定契约** — MCP / HTTP / menubar 都通过 CLI
3. **MCP 永远 spawn CLI** — 不直接调用 HTTP
4. **本地优先** — 云端能力（Claude fallback）必须显式 opt-in
5. **LLM 失败不阻塞** — 确定性步骤（chunk/redact/FTS5）永远执行，LLM 增强步骤失败时降级

## 目录结构规划

```text
memex/
├── Cargo.toml              # workspace root
├── crates/
│   ├── memex-core/         # 核心逻辑（collector/processor/storage/retriever）
│   ├── memex-cli/          # CLI 入口（clap），内含 `memex mcp` 子命令
│   ├── memex-mcp/          # MCP server 实现（rmcp，被 memex-cli 调用）
│   └── memex-daemon/       # HTTP daemon（axum + file watcher）
├── tauri-app/
│   ├── src-tauri/          # Tauri 后端（IPC commands）
│   └── src/                # Vue 3 前端（menubar UI）
└── tests/                  # 集成测试
```
