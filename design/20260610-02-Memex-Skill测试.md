# Memex SKILL 自然语言测试用例 + 实测报告

> **日期**：2026-06-10
> **被测对象**：`memex skill` / `memex skill-status` CLI（部署版 v0.3.4，`/Applications/Memex.app/Contents/MacOS/memex`）
> **覆盖范围**：4 个 IDE 的 SKILL.md 投递（Cursor / Claude Code / Codex / OpenCode）
> **方式**：read-only 检查 + 内容比对（不跑 uninstall，避免破坏用户已配置环境）

---

## 测试用例（自然语言描述 + 预期）

### 1. CLI 接口

**1.1 `memex skill-status` 列出全部 4 个 IDE**
- 行为：直接跑命令，无参数
- 预期：返回 4 行，分别对应 cursor / claude-code / codex / opencode，每行带 `[✓]`、安装路径、字节数

**1.2 单 IDE 查询 `memex skill cursor --status`**
- 行为：用 `--status` flag 查单个 IDE
- 预期：返回与 1.1 中 cursor 那一行一致的状态信息，installed=true

**1.3 未知 IDE 报错**
- 行为：`memex skill made-up-ide --status`
- 预期：CLI 返回非零退出码 + 错误说明（unknown ide / target 不在白名单）

### 2. 文件落点

**2.1 4 个目标路径全部已安装**
- 行为：直接 `ls -la` 4 条 dest path
- 预期：4 条文件全部存在，字节数与 `memex skill-status` 报告一致

**2.2 仓库源文件 ↔ IDE 安装文件字节相等**
- 行为：`cmp skills/<ide>/SKILL.md <ide-dest-path>` 4 次
- 预期：4 次全部 silently exit 0（cmp 字节级相等）。这同时验证了 `include_str!` 编译后 binary 写出的内容 == 当前仓库源

**2.3 SKILL.md 内容首部包含 IDE 标识**
- 行为：读每个安装文件的前 6 行
- 预期：YAML front-matter 含 `name: memex`、`client: <对应 ide>`，body 第一段提到该 IDE 名（"Cursor Skill" / "Claude Code Skill" 等）

### 3. 幂等性

**3.1 重新安装已装的 IDE**
- 行为：`memex skill cursor`（不带 `--uninstall`）
- 预期：安装命令成功输出，文件大小保持 2752 bytes 不变。等价于覆写。

### 4. 工具调用样例完整性（SKILL.md 元数据契约）

**4.1 4 个 IDE 的 SKILL.md 都包含「触发短语」段**
- 预期：4 个文件正文末尾都有 `## 触发短语` 章节，列出英文触发词（如 `search memory` / `find session` / `recall` 等）

**4.2 4 个 IDE 的 SKILL.md 都包含「工具调用样例」表格**
- 预期：4 个文件 body 中都有一张表，把"用户原话"映射到具体的 MCP 工具名

### 5. 破坏性操作（人工验证，自动测试不跑）

> 以下用例**不在自动跑**，因为会真删除你的 IDE skill 配置文件。如果想验证 uninstall 流程，按下面步骤手动操作：

**5.1 卸载 + 重装回环**

```bash
# 任选一个 IDE（这里以 codex 为例，路径不影响 Cursor / Claude Code 主用例）
memex skill codex --uninstall
ls -la ~/.codex/prompts/memex.md  # 应当返回 No such file
memex skill-status                 # codex 行应当 [✗] installed=false

memex skill codex                  # 重新安装
memex skill-status                 # codex 应当回到 [✓] 3118 bytes
```

预期：uninstall 后文件消失、status 报 `installed=false`；重装后字节数恢复。

## 实测结果

| # | 用例 | 结果 | 关键证据 |
|---|---|---|---|
| 1.1 | `memex skill-status` 列出 4 个 IDE | **PASS** | 4 行全部 `[✓]`，字节数：cursor=2752 / claude-code=3201 / codex=3118 / opencode=3029 |
| 1.2 | `memex skill cursor --status` 单 IDE | **PASS** | 输出 `cursor: installed=true, path=/Users/TerrellShe/.cursor/skills/memex/SKILL.md, size=Some(2752)`，与 1.1 中 cursor 行字节数一致 |
| 1.3 | 未知 IDE 报错 | **PASS** | 退出码 `exit=1`，stderr=`Error: Unknown IDE: made-up-ide. Supported: cursor, claude-code, codex, opencode` |
| 2.1 | 4 个目标路径全部存在 | **PASS**（≡1.1） | `skill-status` 已经做完全等价的检查，字节数互相吻合 |
| 2.2 | 仓库源 ↔ IDE 安装文件 cmp | **PASS** | `cmp -s` 4 次全 silent exit 0：cursor / claude-code / codex / opencode 全 EQUAL；说明 v0.3.4 binary `include_str!` 编译时锁定的内容与当前仓库源一致 |
| 2.3 | YAML front-matter 含 `name: memex` + `client: <ide>` | **PASS** | 4 个文件首 6 行结构相同，client 字段分别是 cursor / claude-code / codex / opencode |
| 3.1 | 重装幂等性 | **PASS** | re-install cursor 前后 size 都是 2752，输出 `cursor skill installed at ...`，无副作用 |
| 4.1 | `## 触发短语` section 存在 | **PASS** | 4 个 SKILL.md 都有 6 个 `^## ` 顶级章节，触发短语 section 命中 |
| 4.2 | 「用户原话 → 应调用」表格存在 | **PASS** | 4 个文件都包含「用户原话」表格，每个都 3~4 次提及 `search_memory` |

### 总结

- **9 / 9 用例全部 PASS**
- 4 个 IDE 的 SKILL.md 投递机制（`include_str!` 嵌入 → fs::write 写出）在 v0.3.4 部署版上工作正常
- read-only 与幂等写均符合预期，破坏性操作（uninstall）按 5.1 的脚本可由用户手动验证
- 仓库源文件 `skills/<ide>/SKILL.md` 与所有 IDE 安装位置字节级一致 → 说明 binary 与仓库 sync，没有"装的是旧版本"的滞后问题

### 与 MCP 测试（[20260610-01](20260610-01-Memex-MCP测试.md)）的协作关系

- **MCP** = 运行时协议层。AI 调用工具时实时通过 stdio JSON-RPC 拿数据。
- **Skill** = 配置层。把 SKILL.md 投递到 IDE 的 skills 目录，让 AI 在合适语境下"知道有 memex 这个工具可调"。
- 二者关系：**Skill 引导 AI 学会用 MCP**。Skill 装好但 MCP 没接通 → AI 知道 memex 存在但调不出来；MCP 接通但 Skill 没装 → AI 不知道在什么场景下应该用 memex（除非用户每次主动提"用 memex 帮我搜..."）。两者都装才形成闭环。

## 你装好 Skill 之后，应该怎样在 IDE 里测试触发？

> 上面 9 个用例验证了**安装机制层**（开发者视角）。下面这一节是**最终使用者视角**：你在 IDE 里实际打字时，怎么验证 SKILL.md 真的被 AI 加载并触发了 memex MCP。

### A. Cursor 中的触发测试

**前提**：Cursor 已通过 `memex setup cursor` 配好 MCP，重启过 Cursor。

**第一步：直接显式触发**

```
@memex 帮我搜下之前讨论 "Tauri IPC camelCase" 的 session
```

或者 Composer 里直接说：

> 用 memex 搜一下我之前聊过 reasoning model max_tokens 的会话

预期：AI 调 `search_memory(query="reasoning model max_tokens")`，回答里出现 `memex://session/<uuid>` deep link。

**第二步：隐式触发（更能反映 SKILL.md 是否真正引导）**

```
我之前是怎么解决 SQLite WAL 模式问题的？
```

按 `skills/cursor/SKILL.md` 的「适用场景」明确写了 *"我之前是怎么解决 X 的？"*——AI 应当主动调 `search_memory`，而不是说"我不知道你之前做了什么"。

```
前几天讨论的 Redis pipeline 方案最后定了哪个？
```

——这条问法在 SKILL.md 里被列为典型示例，AI 应该自动触发 memex。

**判断 SKILL.md 加载了的标志**：
- 在多个 SKILL.md 里出现的同一个触发词（如 "what did I discuss" / "我之前讨论过"）能让 AI 主动调 MCP，而不是回 "I don't have that history"
- 回答里出现具体的 session id 或 deep link，而不是凭空编造

### B. Claude Code 中的触发测试

**前提**：`memex setup claude-code` 完成，重启 Claude Code。

Claude Code 的工具调用前缀是 `mcp__memex__*`，所以触发词需要更明确：

```
使用 mcp__memex__search_memory 帮我找之前讨论 zoom-docs 的会话
```

或者隐式：

```
我和 Claude Code 在 zoom-docs 项目里聊过什么？
```

```
上次讨论的 retry 策略最终定的是哪个？
```

**判断**：Claude Code Debug 模式（`claude --debug`）启动时能看到 tool_use JSON-RPC 调用日志中包含 `mcp__memex__search_memory`。

### C. Codex 中的触发测试

**前提**：`memex setup codex` 完成，重启 Codex CLI（或 IDE 集成）。

Codex 的 SKILL 是通过 `prompts/<name>.md` 提供的 slash command。所以测试入口是：

```
/memex
```

slash command 触发后，Codex 会读取 `~/.codex/prompts/memex.md` 的内容作为 system prompt 注入，AI 就知道接下来要调用 memex MCP 工具。

### D. OpenCode 中的触发测试

**前提**：`memex setup opencode` 完成，重启 OpenCode。

OpenCode 的 SKILL 也是 slash command 形态：

```
/memex
```

之后再问类似 *"找一下我之前聊过的 docker compose 配置"* 即可。

### E. 跨 IDE 的"5 分钟验收流程"

照下面 5 步走，5 分钟内能验收 4 个 IDE 的 Skill 装载是否到位：

| Step | IDE | 输入 | 期望 |
|---|---|---|---|
| 1 | Cursor Composer | "我之前是怎么解决 X 的？"（X 替换为你真做过的话题） | AI 主动调 `search_memory` 并返回 deep link |
| 2 | Claude Code | "我之前讨论过什么决策？" | AI 调 `mcp__memex__search_memory` |
| 3 | Codex | `/memex` 然后问 "我跟 codex 聊过哪些 session" | AI 主动调 `list_recent` |
| 4 | OpenCode | `/memex` 然后问 "我有多少 memex 索引数据" | AI 主动调 `stats` |
| 5 | 任意 IDE | "帮我写一份本周工作总结" | AI 调 `list_sessions_by_range` 拿 7 天数据 |

如果 5 个 step 里任意一个 AI 回答 "I don't have access to previous conversations" 或类似话术——大概率：
- MCP server 没接通 → 跑 `memex doctor` 检查
- SKILL.md 没装到该 IDE → 跑 `memex skill <ide>` 重装
- IDE 没重启 → 关掉重开（特别是 Claude Code 和 Codex 需要重启进程）

### 排查清单

| 症状 | 可能原因 | 验证命令 |
|---|---|---|
| AI 不知道 memex 是什么 | SKILL.md 没装 | `memex skill-status` 看是不是 [✗] |
| AI 知道 memex，但调不出工具 | MCP server 没注册到 IDE | `memex setup <ide>` 重做一次，重启 IDE |
| AI 工具调用了但返回 0 结果 | 数据库为空 / 还没采集 | `memex stats` 看 sessions 数；为 0 就 `memex ingest` |
| 调出来了但只返回 1~2 条 | 索引未完全 / 采集中 | 等待 daemon 完成；或手动 `memex ingest --adapter cursor` |
