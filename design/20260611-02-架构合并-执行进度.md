## Phase 1 — daemon lib 暴露 `run_in_process` ✅

**Commit**：`b3cda45 refactor(daemon): expose run_in_process() entry for in-process hosting`

**改动**：
- `crates/memex-daemon/src/lib.rs`：
  - 拆分 `run(port)` 为两层：
    - 外层 `run(port)`：handle lockfile + logger + SIGTERM handler，调内层
    - 内层 `run_in_process<F: Future<Output=()> + Send + 'static>(memex_dir, db, port, shutdown)`：watcher + bootstrap ingest + axum server，不写 lockfile / 不装 signal handler
- `crates/memex-daemon/src/tests.rs`：
  - 新增 `test_run_in_process_obeys_external_shutdown` —— 验证 in-process 入口必须接受外部 shutdown future + 必须不写 lockfile

**验证**：
- `cargo test -p memex-daemon --lib` → 11/11 pass
- `cargo build -p memex-daemon --release` → binary 10.9 MB 正常产出
- binary 启动路径完全没改，旧调用方零感知

**未做（留给后续 Phase）**：
- Tauri 主进程**没**实际调用 `run_in_process` —— 此 phase 只暴露 API
- daemon binary 仍是 bundle 的 sidecar（`Contents/MacOS/memex-daemon` 在）

## Phase 2 — Tauri 内嵌 daemon ⬜

**目标**：Tauri builder.setup() 里 `tauri::async_runtime::spawn(memex_daemon::run_in_process(...))`，commands/daemon.rs 从 spawn 子进程改成读 in-process state。

**TODO**：
- [ ] `tauri-app/src-tauri/src/lib.rs`：setup() 里启动 daemon
  - 创建 shutdown channel（`tokio::sync::watch` 或 `Notify`）
  - spawn `run_in_process`，把 JoinHandle 和 shutdown trigger 存进 `tauri::State`
  - daemon HTTP port：默认 9999；冲突时（绑端口失败）选个 free port 然后写到 `tauri::State::DaemonPort(u16)` 让前端 + commands 都能查
- [ ] `tauri-app/src-tauri/src/commands/daemon.rs`：
  - `daemon_status()` → 不再读 daemon.lock，直接查 `tauri::State<DaemonHandle>`
  - `daemon_restart()` → 不再 fork 子进程，trigger 旧 shutdown + spawn 新的 run_in_process
  - `stop_daemon_blocking()` → 改成 trigger shutdown channel + await join handle，移除 `kill -TERM/-KILL` 子进程逻辑
- [ ] `RunEvent::ExitRequested` 处理：drop daemon trigger，确保 graceful shutdown
- [ ] 一个集成测试覆盖：Tauri builder 启动后 GET /health 应该 200，且不写 daemon.lock

**预估工作量**：2-3 小时
**风险**：tokio runtime 跟 tauri 同步 / daemon 异常时不能 crash 整个 GUI
