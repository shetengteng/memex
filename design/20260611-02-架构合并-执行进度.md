### Phase 5b — daemon 补 endpoint + 剩余数据命令切 RPC（缩窄版）

**用户决策（2026-06-11 19:10）**：原计划"全切 RPC"被否决。理由是 `context`/`reflect` 走 RPC 后 hook 高频路径会被 daemon 关机静默拖死，`rebuild/backup/restore` 是 admin 级危险操作（本来就要先停 daemon），强行 RPC 化反而增加 30s timeout 误杀风险，得不偿失。

**5b 实际只做 `stats` 和 `ingest` 两条**：

| 子阶段 | 内容 | 状态 |
|---|---|:-:|
| 5b-2 | daemon `/stats` 扩 `today` + `last_7_days`；CLI `stats` 切 RPC（删 db 直连） | ✅ |
| 5b-3 | daemon 新增 `POST /ingest`；CLI `ingest` 切 RPC（用 `post_long` 10 min timeout） | ✅ |

**保留本地直连的命令（决策已落，不再迁移）**：

| 命令 | 保留原因 |
|---|---|
| `context` | IDE hook 高频路径（SessionStart 每次都跑）。daemon 关机时 hook 仍需可用，否则 hook 静默挂掉拖累 IDE 启动 |
| `reflect run/list/show` | 内部跑 LLM，单次可能数十秒。HTTP 长连 + 进度反馈机制复杂；这条命令本来就 manual 触发不高频 |
| `rebuild-index` | admin 级危险操作。本来就要先停 daemon 才能跑（SQLite WAL 不允许并发写），RPC 化反而要绕一圈 |
| `backup <path>` | 纯文件操作（tar.gz 打包 `~/.memex`），跟 db 没耦合，没有走 daemon 的理由 |
| `restore <path>` | 同上 + 强烈建议从 Tauri UI 调（已有 `import_db` IPC 包好 stop→restore→start 事务） |
| `config show/set` | 已决策保留本地 toml |
| `doctor` | 本来就是本地诊断（lock / db / dirs 状态检查），是 daemon 没跑时的兜底入口 |

#### 5b-2 改动详情

- daemon `routes::stats` 现在返回：
  ```json
  { "sessions": N, "messages": N, "chunks": N, "sources": N,
    "today": { "search_count": K, "mcp_calls": K, "ingest_messages": K, ... },
    "last_7_days": { ... } }
  ```
  `today` 来自 `db.get_today_metrics()`，`last_7_days` 来自 `db.get_metrics_range(7)` 求和。空 db 时 today/last_7_days 是 `{}` 不是 `null`，避免 CLI 解析坑。
- CLI `commands/stats.rs` 完全重写：删了所有 `Db::open` / metrics 模块依赖，只通过 `MemexClient::get_value("/stats")` 拿 JSON，按 `METRIC_LABELS` 表格打印。
- daemon 端新增 2 个测试：`test_stats_zero_for_empty_db`（断言 today/last_7_days 是空对象）、`test_stats_aggregates_today_and_last_7_days`（写入 search/ingest/mcp metric，断言两个 bucket 都聚合到正确数值）。

#### 5b-3 改动详情

- daemon `routes::ingest` (POST /ingest)：body `{ adapter?: string }`，`spawn_blocking` 跑 `memex_core::ingest::run_ingest`，避免阻塞 axum runtime；其他端点正常响应。
- `lib.rs` 路由表追加 `.route("/ingest", post(routes::ingest))`，`use axum::routing::{get, post}`。
- CLI `client.rs` 新增 `LONG_REQUEST_TIMEOUT = 10 min` 和 `long_agent`；`post_long(path, body)` 公开 API。
- CLI `commands/ingest.rs` 完全重写：删 `memex_core::ingest` / `Db::open` 依赖，走 `client.post_long("/ingest", ...)`。
- daemon 端新增 `test_ingest_returns_zero_on_empty_home` —— 把 `HOME` 重定向到 tempdir 跑空 ingest，断言 200 + 0 messages/chunks。
