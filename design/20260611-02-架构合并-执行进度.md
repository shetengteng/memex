## Phase 1 — daemon lib 暴露 `run_in_process` ✅

**Commit**：`b3cda45 refactor(daemon): expose run_in_process() entry for in-process hosting`

**改动**：
- `crates/memex-daemon/src/lib.rs`：
  - 拆分 `run(port)` 为两层：
    - 外层 `run(port)`：handle lockfile + logger + SIGTERM handler，调内层
    - 内层 `run_in_process<F: Future<Output=()> + Send + 'static>(memex_dir, db, port, shutdown)`：watcher + bootstrap ingest + axum server，不写 lockfile / 不装 signal handler
- `crates/memex-daemon/src/tests.rs`：
  - 新增 `test_run_in_process_obeys_external_shutdown` —— 验证 in-process 入口必须接受外部 shutdown future + 必须不写 lockfile

**验证**：
- `cargo test -p memex-daemon --lib` → 11/11 pass
- `cargo build -p memex-daemon --release` → binary 10.9 MB 正常产出
- binary 启动路径完全没改，旧调用方零感知

**未做（留给后续 Phase）**：
- Tauri 主进程**没**实际调用 `run_in_process` —— 此 phase 只暴露 API
- daemon binary 仍是 bundle 的 sidecar（`Contents/MacOS/memex-daemon` 在）

## Phase 2 — Tauri 内嵌 daemon ✅

**Commit**：待 commit（下方改动完成于本次会话）

**目标**：Tauri builder.setup() 里 `tauri::async_runtime::spawn(memex_daemon::run_in_process(...))`，commands/daemon.rs 从 spawn 子进程改成读 in-process state。

**已完成**：
- [x] `tauri-app/src-tauri/Cargo.toml`：新增 `memex-daemon = { path = "../../crates/memex-daemon", version = "0.3" }` 依赖（之前是 sidecar binary，本 phase 改成 lib 依赖）
- [x] `tauri-app/src-tauri/src/services/mod.rs` + `services/daemon.rs`（新增 ≈140 行）：
  - `DaemonHandle` —— 单次启动的 shutdown_tx + JoinHandle 句柄
  - `DaemonState(Mutex<Option<DaemonHandle>>)` —— Tauri `State` 持有的可变容器，封装 install / shutdown / restart / snapshot
  - `spawn_in_process(port)` —— 调 `memex_daemon::run_in_process` 起 axum，写 `daemon.lock` 但 pid=主进程 PID
- [x] `tauri-app/src-tauri/src/lib.rs::setup()`：
  - 先 `commands::daemon::stop_daemon_blocking()` 清掉历史独立 daemon 的 lock
  - `app.manage(services::daemon::DaemonState::new())` 早注入空容器，让 IPC 立刻可查
  - 异步 `services::daemon::spawn_in_process(DEFAULT_PORT)` → `state.install(handle).await`
- [x] `tauri-app/src-tauri/src/lib.rs::RunEvent::ExitRequested`：
  - 优先 `state.shutdown_blocking()`（trigger oneshot + await join + remove lock）
  - 没 state（极少见）→ 旧的 `stop_daemon_blocking()` 兜底
- [x] `tauri-app/src-tauri/src/commands/daemon.rs`：
  - `daemon_status(state)` → 直接读 `state.snapshot()`，再调 HTTP `/health` 探活
  - `daemon_restart(state)` → 调 `daemon_restart_inner(&state)`：`state.restart(port) + 轮询 HTTP 就绪`
  - `daemon_restart_inner(&DaemonState)` —— 抽出复用入口（backup/maintenance 都用）
  - 删除 `find_daemon_binary` —— 不再 spawn 独立子进程
  - `stop_daemon_blocking()` 已带 self-pid 守卫（Phase 1.5 已加），lock 文件指向自己时只清不杀
- [x] `tauri-app/src-tauri/src/commands/backup.rs::import_db`：签名加 `state: tauri::State<'_, DaemonState>`，路径校验抽 `validate_import_source`（测试改测它）
- [x] `tauri-app/src-tauri/src/commands/maintenance.rs::restart_after_reset`：通过 `app.state::<DaemonState>()` 拿引用调 `daemon_restart_inner`

**验证**：
- `cargo clippy -p memex-menubar --all-targets` → 通过
- `cargo test -p memex-menubar` → 33/33 pass（19 lib + 11 ipc_contract + 3 window_config）

**未做（留给后续 Phase）**：
