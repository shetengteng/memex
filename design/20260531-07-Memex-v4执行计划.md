## 2. v0.0 Validation MVP

### Sprint 1：骨架 + Claude adapter + Storage（W1-2）

**骨架**
- [ ] 初始化 `memex` CLI 项目（pyproject.toml / Click / 目录布局）
- [ ] 实现 `~/.memex/` 工作目录自动创建
- [ ] 实现 `config.toml` 默认配置生成
- [ ] 实现 `redactions.yaml` 默认脱敏规则文件生成
- [ ] 实现统一错误类型 + CLI JSON 输出格式（`--json` flag）

**Claude Code Adapter**
- [ ] 扫描 `~/.claude/projects/` 发现项目目录
- [ ] 解析 `<session-uuid>.jsonl` 消息格式
- [ ] 实现按 mtime + offset 增量读取（不重复处理已读消息）
- [ ] 实现 session 边界判定（同文件 = 同 session）

**Storage**
- [ ] 实现 SQLite schema 初始化（sources / sessions / messages / chunks 表）
- [ ] 实现 `chunks_fts` FTS5 虚拟表 + content sync trigger
- [ ] 启用 WAL 模式
- [ ] 实现 Markdown session 文件写入（YAML frontmatter + 消息体）
- [ ] 实现写入顺序保证：MD 先写 → SQLite 后写 → offset 最后更新

**验收**
- [ ] `memex ingest` 从真实 Claude Code 历史读取 ≥ 10 条消息
- [ ] 生成的 Markdown 文件可直接打开阅读
- [ ] `SELECT * FROM chunks_fts WHERE chunks_fts MATCH 'redis'` 返回结果

### Sprint 2：其他 3 Adapter + Processor（W3-4）

**Cursor Adapter**
- [ ] Spike：确认 `state.vscdb` 路径、SQLite 表结构、JSON blob 格式
- [ ] 实现 Cursor adapter（只读 SQLite → 提取 composerData）
- [ ] 处理 macOS Full Disk Access 权限缺失时的友好报错

**Codex Adapter**
- [ ] 实现 Codex adapter（`~/.codex/sessions/<uuid>/session.jsonl`）
- [ ] 增量 offset 记录

**OpenCode Adapter**
- [ ] Spike：确认 `~/.opencode/sessions/*.json` 格式
- [ ] 实现 OpenCode adapter

**Processor**
- [ ] 实现 Markdown 结构切分（标题 / 代码块 / 段落 → chunk）
- [ ] 实现 `chunk_type` 识别（tool_call / code_block / text / assistant_turn）
- [ ] 实现内置 redaction 规则（email / phone / API key / IP / 银行卡号）
- [ ] 实现自定义 redaction 规则从 `redactions.yaml` 加载
- [ ] 实现 metadata 自动抽取（topics / languages / has_code / tools_used / error_keywords）
- [ ] 实现 `sources` 表记录每个 adapter 扫描状态

**验收**
- [ ] 4 个 adapter 中 ≥ 3 个能采集真实数据
- [ ] chunk_type 识别准确率 > 90%（抽样 50 条人工验证）
- [ ] redaction 对 `sk-ant-xxx` 格式 100% 命中

### Sprint 3：Retriever + CLI 完整（W5-6）

**检索**
- [ ] 实现 FTS5 BM25 chunk 级检索
- [ ] 实现 chunk → message → session JOIN 回溯
- [ ] 实现结果返回字段：snippet / timestamp / adapter / session_url
- [ ] 实现 ranking：BM25 + recency_boost + project_match_boost
- [ ] 实现 `match_reason` 字段生成（命中位置说明）
- [ ] Spike：中文分词方案选型（unicode61 vs jieba vs bigram）

**CLI 命令**
- [ ] `memex ingest` — 触发全量/增量采集
- [ ] `memex process` — 触发 processor 处理队列
- [ ] `memex search "query"` — 返回结果列表
- [ ] `memex search "query" --json` — 输出稳定 JSON 格式
- [ ] `memex sessions list --days N` — 按时间范围列出 session
- [ ] `memex sessions show <id>` — 展示单个 session 详情
- [ ] `memex stats` — 输出统计信息（session 数 / message 数 / chunk 数）
- [ ] `memex config get/set` — 配置项读写
- [ ] `memex backup <path>` — 备份 sessions/ 目录
- [ ] `memex rebuild-index` — 从 Markdown 重建 SQLite 索引

**验收**
- [ ] `memex search "redis pipeline"` 在 < 100ms 内返回正确结果
- [ ] `memex search --json` 输出含 snippet + match_reason
- [ ] `memex rebuild-index` 后搜索结果与原始一致

## 4. v0.2 menubar PMF

### Sprint 7：daemon + HTTP（W13-14）

- [ ] 实现 Python daemon 常驻进程（graceful start/stop）

- [ ] Collector 改为 daemon 内 file watcher（watchdog 库）

- [ ] `access_log` 表写入（每次搜索记录 query / timestamp / result_count）
- [ ] `metrics` 表日聚合（daily sessions / messages / searches）
