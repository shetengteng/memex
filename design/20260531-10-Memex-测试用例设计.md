# Memex v4 测试用例设计

> 日期：2026-05-31
> 关联：`20260531-07-Memex-v4执行计划.md`
> 用途：定义每个模块的测试策略、用例分类和验收标准。每个 Sprint 完成功能后按此文档编写和执行测试。

---

## 1. 测试分层策略

| 层级 | 覆盖范围 | 工具 | 执行时机 |
|---|---|---|---|
| 单元测试 | 单个函数/方法 | pytest (Python) / cargo test (Rust) | 每次提交 |
| 集成测试 | 模块间交互 | pytest + fixture / Rust integration test | 每 Sprint |
| 契约测试 | CLI JSON 输出格式 | snapshot test | 每次修改 CLI 输出 |
| E2E 测试 | 完整用户流程 | shell script + assert | 每 Sprint 末 |

## 4. 性能基准

| 指标 | 目标 | 测量方法 |
|---|---|---|
| 单次 ingest 100 条消息 | < 2s | `time memex ingest` |
| FTS5 搜索 10k chunks | < 100ms | `time memex search "query" --json` |
| MCP 冷启动（v0.3） | < 25ms | subprocess spawn to JSON output |
| Popup 打开延迟 | < 100ms | Tauri IPC round-trip |
| rebuild-index 128 sessions | < 5s | `time memex rebuild-index` |
