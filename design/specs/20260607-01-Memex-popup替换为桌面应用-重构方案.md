# Memex Popup → 桌面应用 重构方案

> 日期：2026-06-07
> 状态：待执行（不考虑兼容性，一次性切换）
> 范围：用 `design/prototype-app/` 的最新 Vue 原型**完全替换**当前 `tauri-app/` 的 popup + dashboard 双形态前端，改造为单桌面应用 + 托盘极简弹窗
> 不动：`crates/memex-core` / `crates/memex-daemon` / MCP 协议 / 数据模型 / IPC contract（`invoke` 命令名与参数）

---

## 一、决策摘要（基于用户确认）

| 决策项 | 选择 | 含义 |
|---|---|---|
| 应用形态 | **主窗口 + 托盘极简弹窗** | 主窗口承担全部功能（5 大模块）；托盘左键 ≈360px 极简卡片（最近会话 + 跳板） |
| 关闭按钮（红色圆点） | **缩到托盘** | 隐藏主窗口，daemon 持续运行，托盘图标继续可见 |
| Dock 策略（macOS） | **动态** | 有窗口可见时显 Dock 图标；全部窗口隐藏后退回 Accessory（菜单栏应用模式） |
| 全局快捷键 `⌘⇧M` | **切换主窗口** | 显示则隐藏，隐藏则显示并聚焦 |
| 兼容性 | **不考虑** | 删除旧 popup 视图、旧 dashboard sidebar、deep link 兼容路径直接迁移 |

### 3.4 Rust 侧关键改动

#### 3.4.1 启动激活策略（`lib.rs::run` setup）

```rust
#[cfg(target_os = "macos")]
{
    // 由 update_activation_policy 在窗口可见性变化时动态切换：
    // - 有 main 可见 → Regular（Dock 显示）
    // - 全部隐藏    → Accessory（Dock 隐藏，菜单栏模式）
    app.set_activation_policy(tauri::ActivationPolicy::Regular);
}
```

> 同时 `Info.plist` **删除 `LSUIElement=true`**（否则永远不显示 Dock）。

#### 3.4.2 关闭按钮拦截（主窗口）

```rust
let main = app.get_webview_window("main").unwrap();
let handle = app.handle().clone();
main.on_window_event(move |event| {
    if let tauri::WindowEvent::CloseRequested { api, .. } = event {
        api.prevent_close();
        if let Some(win) = handle.get_webview_window("main") {
            let _ = win.hide();
        }
        update_activation_policy(&handle);
    }
});
```

#### 3.4.3 动态 Dock 切换（macOS）

新增 `fn update_activation_policy(app: &AppHandle)`：

```rust
#[cfg(target_os = "macos")]
fn update_activation_policy(app: &AppHandle) {
    let any_visible = app
        .webview_windows()
        .values()
        .any(|w| w.is_visible().unwrap_or(false) && w.label() != "tray-popup");
    let policy = if any_visible {
        tauri::ActivationPolicy::Regular
    } else {
        tauri::ActivationPolicy::Accessory
    };
    let _ = app.set_activation_policy(policy);
}
```

调用点：主窗口 show / hide / close request 后；tray show/hide 不算（tray-popup 不影响 Dock）。

#### 3.4.4 托盘行为（`tray.rs` 重写）

- 左键 Down → toggle `tray-popup` 显隐 + 跟随图标定位（沿用现有 popup 定位逻辑，宽度改 360）
- 右键 → 原生菜单：
  - `Show Memex` → 显示主窗口 + 聚焦 + 切 Dock
  - `Sessions: N`（仍 10s 一刷）
  - `Search…` → 显示主窗口 + emit `open-command-palette`
  - `Settings…` → 显示主窗口 + 导航到 `/settings`
  - `Quit Memex` → `app.exit(0)`

#### 3.4.5 全局快捷键 `⌘⇧M`

```rust
// 行为变更：toggle main，不再 toggle popup
if let Some(win) = app.get_webview_window("main") {
    if win.is_visible().unwrap_or(false) {
        let _ = win.hide();
    } else {
        let _ = win.show();
        let _ = win.set_focus();
    }
}
update_activation_policy(&handle);
```

#### 3.4.6 失焦不再隐藏

- 删除 `appWindow.onFocusChanged` 中"失焦自动隐藏"逻辑（这是 popup 时代行为）
- 主窗口正常多窗体行为；托盘 popup 仍然失焦隐藏（用户期望）

#### 3.4.7 deep link 处理

- 不再"deep link 触发开 dashboard"那段（dashboard 已合并到 main）
- 改成：deep link → show main + 路由 push `/library?session=<id>`，通过 `emit('deep-link', url)` 让前端 router 处理

### 3.6 入口拆分（main vs tray-popup）

Tauri 两个窗口共用一份 `dist/index.html`，靠 URL hash 区分入口：

- 主窗口加载 `index.html#/today`（默认重定向）
- 托盘 popup 加载 `index.html#/tray-popup`

`App.vue` 根据 `useRoute().path` 判断：
- 命中 `/tray-popup` → 渲染 `TrayPopup` 极简组件（不挂 SidebarProvider）
- 其它路由 → 渲染主窗口完整 shell（Sidebar + Header + RouterView + CommandPalette + Toaster）

或者更干净的做法：分两个 entry html（`index.html` + `tray.html`），各自 `main.ts`。**推荐 hash 路由方案**，避免 vite 多入口配置爆炸。

## 四、不做的事（明确边界）

1. **不改 Rust 核心**：`memex-core` / `memex-daemon` / IPC contract 不动
2. **不改数据库 schema**：所有现有 `invoke` 命令保持原参数原返回
3. **不引入新状态管理库**：继续 composable + ref
4. **不做新功能**（除原型已有 UI）：
   - SmartResume 的 `interruptedAt` 字段（一期降级为最近 N session）
   - Library Threads 真实数据（一期占位）
   - Connect MCP 调用统计（一期占位）
5. **不做兼容**：不保留旧 popup 视图、不保留 dashboard 独立窗口、不保留 menubar-only 模式

| 原型用 `@lucide/vue`，旧代码用 `lucide-vue-next` | 整体改成 `lucide-vue-next`（包名更通用，且旧 `IdeIcon.vue` 已依赖） |
| 原型用 `tailwind@4` + `@tailwindcss/vite`，旧 popup 同样 v4 | 配置一致，直接合并 vite.config |
| 原型用 `vue-sonner` Toaster，旧代码没装 | P1 时加依赖：`vue-sonner` |
| `shadcn-vue Sidebar block` 需要 `SidebarProvider` 包裹，且改根布局 | 原型已经写好了，照搬即可 |
| Tauri 多窗口 hash 路由初始 URL 写法 | 创建窗口时 `WebviewUrl::App("index.html#/tray-popup".into())`，开发模式走 `http://localhost:1520/#/tray-popup` |
| 关闭主窗口拦截 + 退出 app 的差异 | 拦截只对 `CloseRequested` 生效；Quit 菜单 / `Cmd+Q` 走 `app.exit(0)` 不拦截 |
| Dock 切回 Accessory 后再切 Regular 的体验 | macOS 切 ActivationPolicy 会有 1 帧闪烁，主窗口已经 `show()` 后再切 policy，体感可接受；不行就保持 Regular 不变 |
| 托盘 popup 失焦自动隐藏会被主窗口聚焦事件干扰 | 仅在 `tray-popup` 自己 onFocusChanged 中处理；主窗口不挂这个 listener |
| 全局 shortcut `⌘⇧M` 在主窗口已聚焦时再按一次应该隐藏 | 检查 `is_visible() && is_focused()` 才隐藏，否则 show+focus |
| 旧 `dashboard` deep link 路径 `memex://session/<id>` | 主窗口 listen 后由 router push，等同迁移 |
