# Memex v4 单元测试用例设计

> 日期：2026-05-31
> 技术栈：Rust（cargo test）
> 对应执行计划：`20260531-13-Memex-执行计划-Rust-70%.md`

---

## 1. 测试策略

| 层面 | 测试类型 | 工具 |
|---|---|---|
| 模块内逻辑 | 单元测试 | `#[cfg(test)]` + `#[test]` |
| 模块间集成 | 集成测试 | `tests/` 目录 + fixture 数据 |
| CLI 契约 | 端到端 | `assert_cmd` + `predicates` |
| 性能 | benchmark | `criterion` |

### 测试原则

- 确定性步骤（chunk / redact / metadata / FTS5）必须 100% 有测试覆盖
- LLM 依赖步骤通过 mock provider 测试
- 每个 adapter 使用真实 fixture 数据（脱敏后的真实文件格式）
- SQLite 测试使用内存数据库（`:memory:`）
